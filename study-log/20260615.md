# 2026-06-15 Linux运维命令学习日志

## 今日实操命令清单
1. find 全盘检索大体积文件
2. grep 日志关键词过滤，忽略大小写匹配
3. sed 文本全局替换，原地修改文件
4. awk + sort + uniq 日志字段提取与次数统计
5. > 输出重定向，导出日志至独立文件

## 今日踩坑完整记录
1. 权限报错：普通用户读取系统日志、检索根目录，大量出现Permission denied，切换root管理员后解决；
2. sed语法报错：表达式与目标文件之间无空格，触发unknown option报错；
3. 文件名字母拼写错误，命令读取文件失败；
4. 输出重定向前置目录未创建，fault-demo文件夹缺失导致日志导出失败，使用mkdir -p递归创建目录解决；
5. VMware共享文件夹、拖拽图片传输失效，暂时先完成文字文档编写，后续补充截图；
6. SSH密钥配置踩坑：Windows生成的公钥上传GitHub，但Linux虚拟机无配套私钥，认证失败，重新在虚拟机生成ed25519密钥后连通GitHub；
7. 文件权限问题：git克隆仓库后文件归属不是当前用户，普通用户无法写入，使用sudo chown -R 修改目录所有者解决；
8. 文件名空格坑：带空格文件名不加双引号会被识别为多个路径，误创建成文件夹导致无法编辑。

## 今日知识点总结
1. `>` 输出重定向：文件不存在会自动新建，但前置存放目录必须手动创建；
2. sed -i 参数直接修改原文件，生产环境修改配置文件前必须备份；
3. 管道符`|`串联多条命令，前一条命令输出作为后一条命令输入；
4. awk默认以空格分割字段，`$1`代表每行第一列，常用于日志IP提取统计；
5. uniq -c 统计文本重复内容出现次数，日志分析必备工具；
6. 系统级运维排查操作，推荐使用root或sudo临时提权，规避各类权限报错；
7. vim编辑文件出现E212无法写入时，用sudo vim临时获取管理员写入权限。
