# 阿里云Day2 快照备份、磁盘扩容、OSS文件上传实操笔记
## 环境基础信息
- 练习实例：阿里云按量付费ECS CentOS7
- 学习目标：掌握云服务器数据备份、磁盘扩容、OSS文件托管、按需停机省钱，完成JD云平台实操全部要求

## 一、快照备份实操（核心数据备份功能）
### 1. 快照入口位置
![快照菜单入口](./images-day2/01_ecs_snapshot_menu.png)
在ECS实例详情页顶部菜单栏找到【快照和镜像】，快照是针对系统盘/数据盘的整机镜像备份，用于误删文件、系统崩溃时快速恢复数据。

### 2. 创建快照操作
![新建快照表单](./images-day2/02_create_snapshot_form.png)
操作流程：快照列表 → 创建快照
- 快照名称：day2-ecs-system-disk-backup
- 描述：阿里云练习系统盘备份，用于回滚测试
- 快照创建期间服务器不会停机，不影响业务运行

### 3. 快照生成完成状态
![快照完成列表](./images-day2/03_snapshot_finished_list.png)
快照创建耗时随磁盘容量变化，完成后会显示「已完成」状态，快照会单独收取存储费用，长期不用可删除节省成本。

### 4. 快照回滚功能（仅演示不执行）
![回滚确认弹窗](./images-day2/04_snapshot_rollback_pop.png)
快照右侧操作栏选择【回滚磁盘】，注意：
1. 回滚会覆盖当前磁盘全部数据，操作前务必备份重要文件
2. 回滚过程服务器会自动关机，业务会中断，生产环境需错峰操作

## 二、ECS停机与计费规则（省钱实操）
![停止实例计费提示](./images-day2/05_ecs_stop_instance_tip.png)
1. 按量付费实例**停机后CPU、内存停止计费**，仅磁盘、快照、公网IP保留收费
2. 短期不用服务器直接停机，长期不用可释放实例彻底停止扣费
3. 区分「停止」和「释放」：停止可随时开机，释放会销毁整台服务器无法找回

## 三、云磁盘扩容实操
![磁盘扩容配置页](./images-day2/06_cloud_disk_expand_entry.png)
1. 业务磁盘空间不足时，在磁盘页面执行扩容，仅增加磁盘容量，不格式化原有数据
2. 控制台扩容完成后，需要进入Linux系统内执行分区扩展命令，系统才能识别新增空间
3. 扩容仅收取新增容量的费用，无需重新购买服务器

## 四、OSS对象存储上传文件（JD要求实操）
![OSS文件上传成功页](./images-day2/07_oss_upload_file_success.png)
操作流程：进入OSS Bucket → 文件管理 → 上传文件
1. 适用场景：存放项目部署包、业务运行日志、静态图片资源
2. 优势：不占用ECS本地磁盘，支持外网直接访问，支持权限管控
3. 实操：上传本地测试txt日志文件，验证文件上传、在线预览、下载功能

## 五、Linux磁盘配套命令（虚拟机实操）
```bash
# 1. 查看磁盘分区结构
lsblk
# 2. 查看磁盘挂载与占用容量
df -h
# 3. 查看全部硬盘硬件信息
fdisk -l
##命令作用总结
lsblk：直观查看磁盘、分区挂载对应关系
df -h：查看每个分区剩余可用空间，排查磁盘爆满问题
fdisk -l：识别服务器所有物理磁盘、分区大小
